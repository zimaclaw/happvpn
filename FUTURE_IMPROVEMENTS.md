## P1 — Отказоустойчивость

### Fallback сервер

**Описание:** Добавить второй сервер для автоматического переключения при падении основного

**Текущее состояние:**
- Primary: 🇳🇱 Амстердам (cdn3-87.vk-cdnvideo.com:8443)
- Fallback: TBD

**Рекомендуемый fallback:** 🇩🇪 Франкфурт (cdn3-70.vk-cdnvideo.com:8443)
- Географически близко (~600km)
- Стабильная инфраструктура
- Другой дата-центр

**Логика переключения:**
```yaml
health_check:
  interval: 60s
  timeout: 10s
  retries: 3
  
failover:
  delay: 300s  # 5 минут перед переключением
  auto_revert: true  # вернуться на primary при восстановлении
  revert_delay: 600s  # 10 минут проверки стабильности
```

**Оценка:** ~2 часа (скрипт health check + модификация Dockerfile)

## P2 — Мониторинг

### Метрики

**Описание:** Добавить сбор метрик для анализа работы VPN

**Что отслеживать:**
- Uptime (время работы без сбоев)
- Switch counts (количество переключений)
- Latency (задержка до сервера)
- Traffic (использование трафика)

**Инструменты:**
- Prometheus + Grafana (визуализация)
- Node exporter (сбор метрик)
- Custom exporter (метрики Xray)

**Оценка:** ~4 часа

---

### Алертинг

**Описание:** Уведомления при сбоях

**Каналы:**
- Telegram бот
- Email
- Webhook (custom)

**Триггеры:**
- Сервер недоступен > 5 минут
- Переключение на fallback
- Высокая latency (> 500ms)

**Оценка:** ~1 час

## P3 — Автоматизация

### Ротация серверов

**Описание:** Периодическое переключение между серверами

**Варианты:**
- Round-robin (последовательно)
